# Perbandingan HTML vs HTML + CSS

Project ini nunjukin bedanya website yang cuma pake HTML sama yang udah dikasih CSS. Jauh banget hasilnya!

## Struktur Project

```
Pure HTML vs HTML with CSS Comparison/
├── 1. Pure HTML (Messy)/
│   └── index.html          # HTML aja, ga ada styling
├── 2. HTML + CSS (Clean)/
│   ├── index.html          # HTML yang udah rapi
│   └── styles.css          # File CSS buat styling
└── README.md               # File ini
```

## Perbedaan yang Keliatan

### 1. Pure HTML (Berantakan)

- **Ga ada styling**: Cuma pake style default browser
- **Layout jelek**: Semua elemen numpuk vertikal
- **Ga ada hierarki visual**: Semua text keliatan sama
- **Navigation acak-acakan**: Link cuma text biasa
- **Ga ada warna**: Semua hitam putih doang
- **Ga responsive**: Ga bisa di mobile

### 2. HTML + CSS (Rapi)

- **Styling keren**: Ada warna, font, spacing yang proper
- **Layout grid**: Konten tersusun rapi dalam grid
- **Hierarki jelas**: Font size dan weight beda-beda
- **Navigation bar**: Ada navbar yang proper dengan hover effect
- **Color scheme**: Warna-warna yang konsisten dan gradien
- **Responsive**: Bisa di mobile dan desktop
- **Interactive**: Ada hover effect dan animasi

## Cara Liat

1. Buka `1. Pure HTML (Messy)/index.html` di browser - liat yang jelek
2. Buka `2. HTML + CSS (Clean)/index.html` di browser - liat yang bagus
3. Bandingin perbedaannya

## Yang Bisa Dipelajari

Dari project ini bisa tau:

- Pentingnya CSS untuk web development
- Bagaimana styling bisa mengubah user experience
- Bedanya struktur (HTML) dengan presentasi (CSS)
- Prinsip desain web yang modern
- Konsep responsive design

## Teknologi yang Dipake

- **HTML5**: Markup yang semantic dan terstruktur
- **CSS3**: Styling modern dengan flexbox, grid, gradient, dan animasi
- **Responsive Design**: Mobile-first approach dengan media queries
